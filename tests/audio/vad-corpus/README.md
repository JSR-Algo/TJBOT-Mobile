# VAD Calibration Corpus

Collection of labeled audio samples used to calibrate and validate the
energy + zero-crossing-rate (ZCR) VAD algorithm described in plan §5.6 of
`.omc/plans/realtime-voice-cross-platform-stabilization-v2.md`.

---

## Target Distribution

| Category | Count | Description |
|---|---|---|
| `child/` | 40 | Vietnamese child speech (ages 4-10), 16 kHz mono Int16 |
| `adult/` | 60 | Vietnamese adult speech (mixed M/F), 16 kHz mono Int16 |
| `ambient/` | 60 | Ambient noise only (no speech), 16 kHz mono Int16 |
| `mixed/` | 40 | Child or adult speech with background noise present |
| **Total** | **200** | — |

These targets correspond to the "200-sample child-voice + ambient test corpus"
cited in plan §5.6 for the default VAD constants:
`E_THRESHOLD=-42 dB`, `Z_LOW=0.05`, `Z_HIGH=0.45`, `HANGOVER_MS=400`.

---

## Per-Sample Requirements

Each recorded sample must meet:

- **Format:** WAV, 16 kHz sample rate, mono (1 channel), Int16 PCM (16-bit).
- **Duration:** 1–3 seconds per sample. Shorter clips may miss VAD hangover;
  longer clips should be split into multiple samples.
- **Speech content:**
  - `child/`, `adult/`, `mixed/`: contains at least one speech utterance.
  - `ambient/`: no speech — background noise only (fan, street, HVAC, etc.).
- **No clipping:** peak level must stay below 0 dBFS.
- **Companion sidecar:** every `<name>.wav` must have `<name>.json` conforming
  to `VadSampleSidecar` in `types.ts`.

### JSON Sidecar Shape

Generated by `record.sh` at capture time. Enriched by `label.html` after
human review. Full TypeScript type at `types.ts`.

```jsonc
{
  "schemaVersion": "1.0",
  "capturedAt": "2026-04-28T08:00:00.000Z",
  "category": "child",
  "index": 1,
  "baseName": "child_001",
  "sampleRateHz": 16000,
  "channels": 1,
  "bitDepth": 16,
  "durationSec": 1.85,
  "environment": "living room, quiet",
  "speakerAgeGroup": "child",
  "containsSpeech": true,
  "labelState": "unlabeled",   // → "labeled" after label.html review
  "markers": [],               // → filled by label.html
  "labeledAt": null            // → ISO-8601 timestamp set by label.html
}
```

`markers` array after labeling:
```jsonc
[
  {
    "onsetSec": 0.21,
    "offsetSec": 1.44,
    "confidence": "high",
    "notes": "clean single utterance"
  }
]
```

## Workflow

### Step 1 — Record

```bash
cd tbot-mobile/tests/audio/vad-corpus
./record.sh --category=child --duration_s=2
```

`record.sh` uses `sox` to capture from the default input device. It writes
`child/child_NNN.wav` and `child/child_NNN.json` with `labelState: "unlabeled"`.

See `record.sh --help` for options. On Linux replace the macOS `coreaudio`
driver with `alsa`: see comment in the script.

### Step 2 — Label

Open `label.html` in any modern browser (Chrome, Safari, Firefox):

```
open tbot-mobile/tests/audio/vad-corpus/label.html
```

1. Click **Load WAV** and select `<category>/<name>.wav`.
2. Click **Load JSON** and select the matching `<name>.json`.
3. The waveform renders via Web Audio API.
4. Click the waveform to set the **onset** marker (first speech frame).
5. Click again to set the **offset** marker (first non-speech frame after speech).
6. Adjust **confidence** and add **notes** in the form.
7. Click **Export JSON** — save the updated file over the original sidecar.

Repeat for each sample. Set `labelState` to `"rejected"` via the UI if the
sample has clipping or severe noise artifacts; `calibrate.ts` will skip it.

### Step 3 — Calibrate

After at least 50 labeled samples (ideally the full 200), run:

```bash
cd tbot-mobile
npx ts-node --project tsconfig.json tests/audio/vad-corpus/calibrate.ts
```

The script sweeps the VAD parameter grid and writes the optimal parameters
to `tests/audio/vad-corpus/calibration-result.json`. Commit that file.

Acceptance criterion BI7 (plan §13.3): with the corpus complete, the optimal
`HANGOVER_MS` must be ≥ p95 of observed `barge_in.user_resumed` + 500 ms.

---

## Corpus Integrity Rules

- **Never delete entries to pass calibration.** `calibrate.ts` reports how
  many samples were labeled vs. skipped. A shrinking labeled count is a red flag.
- **Commit WAV files under Git LFS** (or reference storage) — do not commit
  raw audio to the main object store.
- **Do not hand-tune `calibration-result.json`** — re-run `calibrate.ts`.
- The `.gitkeep` files in each sub-directory hold the directory structure in
  git before the first recording session.

## VAD Algorithm Reference (plan §5.6)

```
energy_dB > E_THRESHOLD  AND  zcr in [Z_LOW, Z_HIGH]  →  speech frame
hangover: SPEECH state persists for HANGOVER_MS after first non-speech frame
```

Both iOS and Android run VAD on the **post-conversion 16 kHz Int16 buffer**
(after iOS `AVAudioConverter`, after Android `AudioRecord` output). A 200 ms
ring buffer provides pre-roll on both platforms.

Default starting constants (validated against this corpus; re-run `calibrate.ts`
whenever the corpus grows by 20+ samples):

| Constant | Default | Env override |
|---|---|---|
| `E_THRESHOLD` | `-42 dB` | `EXPO_PUBLIC_VOICE_VAD_E_THRESHOLD` |
| `Z_LOW` | `0.05` | `EXPO_PUBLIC_VOICE_VAD_Z_LOW` |
| `Z_HIGH` | `0.45` | `EXPO_PUBLIC_VOICE_VAD_Z_HIGH` |
| `HANGOVER_MS` | `400` | `EXPO_PUBLIC_VOICE_VAD_HANGOVER_MS` |
